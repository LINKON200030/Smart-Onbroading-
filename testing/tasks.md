# Usability Testing Tasks

These tasks are designed based on key pain points identified in user research, including lack of structure, information fragmentation, and low trust in AI-generated responses.



## Task 1: Understand What to Do Next

**Scenario:**  
You are a new employee on your first day. You want to understand what tasks you need to complete.

**Task:**  
Find your priority tasks and identify what you should do next.

**Success Criteria:**  
- User can locate the "Priority Tasks" section  
- User understands the top 3 tasks  
- User can identify the next action (e.g., click "Continue")  

**Insight Tested:**  
- Reduces cognitive overload  
- Provides structured onboarding  


## Task 2: Find Important Information

**Scenario:**  
You need to find where the API documentation or setup guide is located.

**Task:**  
Use the system to find relevant onboarding documentation.

**Success Criteria:**  
- User uses the Knowledge Hub search  
- User finds relevant result (e.g., Frontend Setup Guide)  
- User understands where the information is located  

**Insight Tested:**  
- Solves information fragmentation  
- Tests effectiveness of "single source of truth"  


## Task 3: Ask AI and Verify the Answer

**Scenario:**  
You are unsure about something and decide to ask the AI assistant.

**Task:**  
Ask a question using the AI assistant and verify if the answer is trustworthy.

**Success Criteria:**  
- User opens the AI (floating button)  
- User asks a question  
- User checks the provided sources  
- User understands where the answer comes from  

**Insight Tested:**  
- Builds trust in AI  
- Tests transparency through source links  

---

## Finally

These tasks validate whether the redesigned onboarding system:
- Provides clear structure and reduces confusion  
- Centralises information effectively  
- Improves trust in AI-assisted support
